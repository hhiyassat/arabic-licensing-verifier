# الأساس النظري للصندوق: المسار التوليدي والمسار التحقيقي

> **عقد هذه الوثيقة:** كل ما هو منقول من الوثائق الأربع موضوع تحت عنوان «منقول».
> وكل ما هو استنتاج لازم عنها موضوع تحت عنوان «مستنتج» ومُعلَّم صراحةً.
> وكل غموض في الأصل مُثبَت غموضًا لا مَحسوم. لا إضافة ولا نقص ولا تصرّف في المنقول.

# الجزء الأول: ما هو منقول حرفيًّا في معناه من الوثائق

## ١-١. القانون الإقليدي العام (من الوثيقة ١)

**منقول.** كل ظاهرة عربية `x` لا تدخل النموذج إلا بإحدى صورتين:

```
Covered(x) = LicensedBranch(x) OR GovernedResidual(x)
```

والفرع المرخّص لا يثبت إلا باجتماع هذه العناصر:

```
LicensedBranch(x) =
    Origin(x)
  + Branch(x)
  + EffectiveAttribute(x)
  + NoInvalidatingDifference(x)
  + ConditionFulfilled(x)
  + CauseActive(x)
  + PreventerAbsent(x)
  + TracePreserved(x)
  + ResidualAudit(x)
```

## ١-٢. معادلة الانتقال (من الوثيقة ١، §٨)

**منقول.** كل انتقال عربي يُكتب هكذا:

```
Transition(A → B) =
    Origin A
  + Candidate Branch B
  + Domain D
  + EffectiveAttribute W
  + InvalidatingDifference Test
  + Condition C
  + Cause S
  + Preventer M
  + Trace T
  + Residual R
  + Rank K
```

ولا يُقال إن `B` ثبت إلا إذا: `A` محفوظ، و`B` فرع لا أصل مزيف، و`D` مضبوط، و`W` مؤثر، ولا فرق قادح، و`C` متحقق، و`S` موجود، و`M` منتفٍ، و`T` محفوظ، و`R` مصرّح به، و`K` لا يتجاوز الدليل.

**منقول.** فإذا اختل عنصر، لا ينتقل النظام إلى نتيجة نهائية، بل ينتج واحدًا من:

```
BlockedCandidate  أو  DeferredCandidate  أو  ResidualCandidate
```

## ١-٣. المصبّات الأربعة (من الوثيقتين ١ و٣)

**منقول.** المخرجات الممكنة لأي انتقال محصورة في:

```
Licensed (مرخّص)  |  Blocked (ممنوع)  |  Deferred (مؤجّل)  |  Residual (بقايا)
```

والصيغة النهائية (الوثيقة ١، §١١):

```
ArabicEuclideanGeometry(x) =
    PreservedOrigin + LicensedBranch + EffectiveAttribute
  + QadihDifferenceTest + Condition + Cause + Preventer
  + Trace + Residual + Rank
```

```
إن نجحت:  x → LicensedCandidate
إن فشلت:  x → GovernedResidual
```

## ١-٤. السلسلة الطبقية (من الوثيقتين ١ و٣)

**منقول.** النظام لا ينتقل من اللفظ إلى الحكم خطيًّا، بل عبر طبقات (الوثيقة ٣):

```
Trace → Carrier → Letter/Haraka → Syllable → Surface
      → Root/Ziyada/Tool → Wazn → Masdar/Event → Role
      → Relation → Ifadah → Hukm
```

وكل انتقال محكوم ببوابة:

```
T_i : L_i → L_{i+1} ∪ Residual ∪ Blocked
```

**منقول.** «فإذا غابت الشهادة، لا ينتقل. وإذا بقي لبس، يخفض الرتبة. وإذا ظهر مانع، يمنع الحكم.»

## ١-٥. دالة التشغيل (من الوثيقة ٣)

**منقول.** التفكير التشغيلي يتحقق إذا امتلك النظام دالة:

```
Θ_S(Input, PK₀) → Output
```

بحيث لا يكون المخرج حكمًا إلا بعد المرور بـ:

```
Trace · Identity · Origin · Branch · EffectiveDescription
· QadihCheck · Condition · Sabab · Mani · Relation
· Ifadah · Evidence · Rank · Residual
```

وللعربية تحديدًا:

```
Θ_ar(Input, PK₀) → Hukm ∪ Residual ∪ Blocked
```

## ١-٦. تقسيم المعلومات السابقة (من الوثيقة ٣)

**منقول.** `PK₀` ليست كلها من العربية، وتُقسَّم:

```
PK₀ = PK_ontic + PK_rational + PK_linguistic
    + PK_lexical + PK_contextual + PK_evidential
```

| الطبقة | تحمل |
|--------|------|
| `PK_ontic` | الوجود، الجنس، الصفة، الحدث |
| `PK_rational` | الهوية، عدم التناقض، السبب، الشرط، المانع |
| `PK_linguistic` | الحرف، الحركة، المقطع، الوزن، العامل، الإعراب |
| `PK_lexical` | الوضع، السماع، المعجم |
| `PK_contextual` | المقام، الإحالة، القرائن |
| `PK_evidential` | الدليل، الرتبة |

## ١-٧. بديهية الحامل (من الوثيقتين ٣ و٤)

**منقول (الوثيقة ٣).** `A0 — لا تشغيل بلا حامل منفّذ`:

```
Operation(A_ar) ⇒ Executor
```

العربية تملك **قانون التفكير**، أما **فعل** التفكير فيتحقق عند التنفيذ داخل حامل.

**منقول (الوثيقة ٤).** الحامل ليس معالج بيانات فارغًا، بل عقل مزوّد فطريًّا بنظام تشغيل عقلي (`Built-in OS`) يحوي مقولات: الوجود، الجنس، العلّة، الدليل، الرتبة، المجاز. والعربية حين تعمل في عقل إنسان لا تعمل كشفرة غريبة، بل كتطبيق متوافق مع نظامه الفطري؛ لا تمنح العقل منطقًا لم يكن له، بل توقظ منطقه الفطري وتجعله يعمل وفق قوانين منظمة مرخّصة. «فالعربية فطرة منطوقة، والعقل الفطري عربية صامتة.»

## ١-٨. قانون الإسناد الموسَّع (من الوثيقة ٣)

**منقول.** الإسناد لا يُمنع مطلقًا، بل يُرخَّص بأحد خمسة:

```
Predicate(x,y) ⇒
    GenusCompatibility(x,y)
  ∨ MetaphoricLicense(x,y)
  ∨ CausalLicense(x,y)
  ∨ ContextualLicense(x,y)
  ∨ EllipticRecovery(x,y)
```

أي: «لا إسناد بلا قابلية جنس أو شهادة نقل مجازي/سببي/سياقي/حذفي.»

## ١-٩. ما تثبته الوثيقة ٣ وما تنفيه (الحدود)

**منقول.** المُثبَت:

```
Think_op(A_ar) ∧ Self_sys(A_ar) ∧ Independent_gov(A_ar)
```

**منقول.** المنفيّ صراحةً (النتيجة الحاسمة):

```
تفكير تشغيلي      ≠  وعي إنساني
ذات نظامية        ≠  نفس شاعرة
استقلال حاكمي     ≠  استقلال وجودي مطلق
تصنيف أنطولوجي داخلي ≠ معرفة مطلقة بالواقع
```

# الجزء الثاني: الغموض المُثبَت في الأصل (لا يُحسَم نيابةً عن النص)

## ٢-١. تناقض اتجاه الخريطة (من الوثيقة ٢ / الصورة)

**منقول — غموض قائم.** في الخريطة الكلية وردت قراءتان متعاكستان للاتجاه:

```
العنوان العلوي:   قابلية الكلمة  →  Hukm        (اتجاه صاعد)
القانون الحاكم السفلي:  Hukm/Ifadah/Relation  →  WordCapability   (اتجاه نازل)
```

**هذا الغموض هو منشأ التمييز بين المسارين.** الوثائق لم تحسمه. وعليه فالمسارُ التوليدي قراءةٌ للاتجاه الصاعد، والمسارُ التحقيقي قراءةٌ للاتجاه النازل. كلاهما له سند في الأصل.

# الجزء الثالث: المسار التوليدي — صياغة أمينة

> **تنبيه أمانة:** الوثائق لم تستعمل لفظ «توليدي/تحقيقي» صراحةً. هذان اسمان وصفيّان
> للاتجاهين اللذين وردا في الخريطة (§٢-١). ما يلي تفصيلٌ لما يلزم عن **الاتجاه الصاعد**
> المنصوص عليه في عنوان الخريطة ودالة التشغيل `Θ_ar` في الوثيقة ٣.

## ٣-١. تعريف المسار التوليدي

**مستنتج من §١-٥ و§٢-١.** المسار التوليدي هو تشغيل دالة `Θ_ar` في اتجاهها الصاعد:

```
المدخل:  Input = (كلمة، سياق/مقام، نوع نص؟)  +  PK₀
السؤال:  إلى أي طبقة في السلسلة (§١-٤) تَصعد هذه الكلمة بترخيص؟
المخرج:  أعلى طبقة بلغتها + الرتبة + الأثر
        ∈ { LicensedCandidate(عند طبقةٍ ما), Blocked, Deferred, Residual }
```

## ٣-٢. ما المنقول فيه

- **منقول:** دالة التشغيل `Θ_ar(Input, PK₀) → Hukm ∪ Residual ∪ Blocked` (§١-٥).
- **منقول:** التدفّق الطبقي `Trace → … → Hukm` (§١-٤).
- **منقول:** أن المخرج لا يكون حكمًا إلا بعد المرور بكامل عناصر الانتقال (§١-٢، §١-٥).
- **منقول:** أن بلوغ طبقةٍ دون الحكم ليس فشلًا؛ فالنظام قد يقف عند طبقةٍ مرخّصة
  (يلزم من قوله «إذا بقي لبس يخفض الرتبة» ومن وجود مصبّ `Deferred`).

## ٣-٣. خاصية المسار التوليدي الجوهرية

**مستنتج.** المسار التوليدي **يكتشف** الحكم؛ الحكم ليس معطى في المدخل، بل ناتج عن صعود
الكلمة عبر البوابات. السؤال المحوري: «ماذا تُنتِج هذه الكلمة؟»

# الجزء الرابع: المسار التحقيقي — صياغة أمينة

> **تنبيه أمانة:** هذا تفصيلٌ لما يلزم عن **الاتجاه النازل** المنصوص عليه في
> «القانون الحاكم» أسفل الخريطة (§٢-١)، وعن آليّة المساءلة المنصوصة حرفيًّا في الوثيقة ٣.

## ٤-١. تعريف المسار التحقيقي

**مستنتج من §١-٢، §٢-١، و§٤-٢.** المسار التحقيقي هو ردُّ حكمٍ **مُدَّعى سلفًا** إلى أصله المحفوظ:

```
المدخل:  Input = (كلمة، سياق/مقام، نوع نص؟، حُكم مُدَّعى)  +  PK₀
السؤال:  هل يَصمد هذا الحكم المُدَّعى عند ردّه إلى أصله عبر شهادات الانتقال؟
المخرج:  صامد / منهار + موضع الانهيار + الرتبة المستحقّة فعلًا
        ∈ { Licensed(صامد), Blocked(منهار), Deferred(مخفوض الرتبة), Residual }
```

## ٤-٢. الآلية المنقولة حرفيًّا (الوثيقة ٣، إثبات الاستقلال الحاكمي)

**منقول.** العربية لا تقبل دعوى الخارج لمجرد صدورها عن متكلم. فلو قيل «هذا المعنى يقيني»،
تسأل العربية:

```
Evidence?   Rank?   Scope?   Residual?   Mani?
```

**منقول.** فإن غاب الدليل، لا تسمح باليقين. وإن بقي لبس، تخفض الرتبة. وإن وُجد مانع، تمنع
الحكم. وإن كان الانتقال قفزًا، توقف المسار.

**منقول.** هذه المساءلة هي عين قانون القبول والرفض:

```
Accept_S(x)  ⇔  Licensed_S(x)
Reject_S(x)  ⇔  ViolatesGate_S(x)
```

**منقول (أمثلة المساءلة من الوثيقة ٣).** فلو ادُّعي أن الحركة إعراب، تسأل العربية:
أين العامل؟ هل الكلمة معربة؟ ما الموقع؟ ما العلاقة؟ هل توجد شهادة انتقال؟ — ولو ادُّعي أن
الضمير يعود إلى مرجع، تسأل: أين المرجع؟ سابق أو لاحق أو مقامي؟ هل يوجد لبس؟ هل الإحالة مغلقة
أم باقية؟

## ٤-٣. خاصية المسار التحقيقي الجوهرية

**مستنتج.** المسار التحقيقي **يُحاكِم** الحكم؛ الحكم معطى في المدخل، والوظيفة هي اختبار
مشروعيّته بردّه إلى أصل. السؤال المحوري: «هل هذا الادّعاء مشروع؟»

# الجزء الخامس: العلاقة بين المسارين

## ٥-١. ما تسنده الوثائق

**منقول.** كلا المسارين يستعمل **نفس** عناصر الانتقال (§١-٢)، ونفس البوابات، ونفس
المصبّات الأربعة (§١-٣)، ونفس قانون «لا قفز». الاختلاف في **الاتجاه** فقط (§٢-١)، لا في
الجبر الأساسي.

## ٥-٢. علاقة الاحتواء

> **هذا مستنتج، لا منقول. أُثبته بوضوح لأنه قرار معماري لا نصّ صريح عليه.**

**مستنتج.** بما أن المسارين يتشاركان عناصر الانتقال نفسها (منقول §١-٢)، فإن التحقيقي
هو التوليدي مُقيَّدًا بحكمٍ مُدَّعى في المدخل ومُشغَّلًا للتأكد من وجود مسارٍ مرخّص يُفضي إليه.
أي: التحقيقي = «هل المسار الذي يُنتِج هذا الحكم موجود ومرخّص؟»

**حدّ هذا الاستنتاج:** الوثائق **لا تنصّ** على هذه العلاقة صراحةً؛ هي لازمٌ هندسيٌّ عن
اشتراكهما في عناصر الانتقال. فمن أراد التزام المنقول وحده، فحسبه أن المسارين متّحدا الجبر
مختلفا الاتجاه (§٥-١)، دون الالتزام بعلاقة الاحتواء.

# الجزء السادس: ملخّص العقد لكلا المسارين

## التوقيع المشترك (منقول §١-٥)

```
Θ_ar : (Input, PK₀) → Hukm ∪ Residual ∪ Blocked
```

## التوليدي (الاتجاه الصاعد)

```
Input  = (كلمة، سياق، نوع نص؟)
المخرج = أعلى طبقة مرخّصة بلغتها الكلمة + الرتبة + الأثر
```

## التحقيقي (الاتجاه النازل)

```
Input  = (كلمة، سياق، نوع نص؟، حُكم مُدَّعى)
المخرج = صامد/منهار + موضع الانهيار + الرتبة المستحقّة + الأثر
```

## الثابت الحاكم لكليهما (منقول §١-٢، §١-٤)

```
لا يُخرَج حكمٌ نهائي إلا بعد ترخيص كل بوابة على المسار.
كل فشل يُصنَّف: Blocked أو Deferred أو Residual — لا يُهمَل.
كل نتيجة تحمل رتبة لا تتجاوز دليلها.
```

**خاتمة العقد:** ما سبق هو الأساس النظري كاملًا، مفصولًا فيه المنقول عن المستنتج. لم يُضَف
إلى المنقول حكمٌ ليس فيه، ولم يُحسَم غموضُ الأصل قسرًا. وما لزم استنتاجه للانتقال إلى الكود
عُلِّم بوصفه استنتاجًا، ليبقى لك حقّ ردّه أو قبوله قبل بناء الجبر.
